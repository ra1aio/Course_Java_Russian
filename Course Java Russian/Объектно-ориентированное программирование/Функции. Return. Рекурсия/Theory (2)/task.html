Условия - стражники

В программировании есть понятия условия стражника. Это такие условия, которые прописываются в начале функции и, если они не выполняются, то функция сразу же завершается с помощью return. Давайте разберёмся на пример. Пусть у нас есть код, функция, которая должна принимать в себя три целочисленных аргумента, которые должны представлять из себя стороны прямоугольного треугольника и находить и выводить на экран его радиус вписанной окружности. Для этого в математике есть формула: r = (a + b - c) / 2, где a и b - его катеты, а с - гипотенуза. Плюсом ко всему наша функция должна обильно делиться информацией, что именно пошло не так. А именно могут быть 2 проблемы: треугольника с такими сторонами может не существовать и это может быть не прямоугольный треугольник. Такой код можно написать следующим образом:

static void findRadius(int a, int b, int c){
    //для удобства сделаем так, чтобы в переменной c лежала длина самой длинной стороны.
    if (a > c) {int d = a; a = c; c = d;}
    if (b > c) {int d = b; b = c; c = d;}
    //теперь в пересенной c лежит наибольшее значение из 3 
    //проверим, что такой треугольник существует
    if (c < a + b){
        //проверим, что это прямоугольный треугольник
        if (c * c == a * a + b * b){
            //вот тут всё в порядке, находим радиус по формуле
            double r = (a + b - c) / 2.0;
            System.out.println("Радиус вписанной окружности равен: " + r);
        }else{
            System.out.println("Этот треугольник не прямоугольный");
        }
    }else{
        System.out.println("Треугольника с такими сторонами не существует");
    }
}

Такая функция работает верно, но код "уходит" от начала строки с каждым условием. А их тут всего два. Давайте теперь напишем этот же код, но с условиями - стражниками. 

static void findRadius(int a, int b, int c){
    //для удобства сделаем так, чтобы в переменной c лежала длина самой длинной стороны.
    if (a > c) {int d = a; a = c; c = d;}
    if (b > c) {int d = b; b = c; c = d;}
    //теперь в переменной c лежит наибольшее значение из 3 
    if (c >= a + b){     //первый стражник: проверяет, что такой треугольник существует
        System.out.println("Треугольника с такими сторонами не существует");
        return;
    }
    if (c * c != a * a + b * b){ //второй стражник: проверяет, что треугольник прямоугольный
        System.out.println("Этот треугольник не прямоугольный");
        return;
    }
    //вот тут всё в порядке, находим радиус по формуле
    double r = (a + b - c) / 2.0;
    System.out.println("Радиус вписанной окружности равен: " + r);
}

Такой же код выглядит более простым, структурированным и не "уходит" от начала строки. Каждый стражник проверяет своё условие. Таких стражников можно добавлять сколько угодно. Это говорит о хорошей масштабируемости кода.
